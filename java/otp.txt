# 🧠 1) Student Management System

## 🔍 Concept

Stores and manages student records using an `ArrayList`.

## 🧱 Core Components

* **Class `Student`**

  * Fields: `id`, `name`, `branch`, `marks`
* **ArrayList**

  * Stores multiple student objects dynamically

## ⚙️ Operations

### ➕ Add Student

* Takes input
* Creates object
* Adds to list
  👉 Important: Duplicate ID check prevents data corruption

### 📋 Display

* Loops through list
* Prints each student

### 🔎 Search

* Linear search by ID
  👉 Time complexity: O(n)

### ✏️ Update

* Finds student → modifies fields

### ❌ Delete

* Removes object using iterator or condition

### 💾 File Handling (advanced)

* Uses `ObjectOutputStream` to save data
* Uses `ObjectInputStream` to load data
  👉 Makes data persistent

# 💳 2) ATM System

## 🔍 Concept

Simulates basic banking operations with authentication.

## 🧱 Core Variables

* `PIN`
* `balance`

## ⚙️ Flow

### 🔐 PIN Verification

* 3 attempts allowed
  👉 Prevents brute force (basic level)

### 💰 Balance Check

* Just prints balance

### ➕ Deposit

* Adds money
* Validates positive amount

### ➖ Withdraw

* Checks:

  * amount > 0
  * amount ≤ balance
    👉 Prevents overdraft

# 🔐 3) Password Validation

## 🔍 Concept

Checks password strength using logic (or regex).

## 🧱 Rules

* ≥ 8 characters
* ≥ 1 uppercase
* ≥ 1 lowercase
* ≥ 1 digit

## ⚙️ Logic

* Loop through characters
* Use:

  * `Character.isUpperCase()`
  * `Character.isLowerCase()`
  * `Character.isDigit()`

👉 Returns boolean + prints missing conditions

* Adds course to list

### ❌ Drop Course
